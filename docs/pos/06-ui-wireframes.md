# 06 — Інтерфейс касира: wireframes

Desktop-first 1920×1080, керування клавіатурою і сканером; миша — допоміжна.
Референс — Торгсофт (функціонально), але без перевантаженості: іконки з підписами,
фіскальні і управлінські операції візуально розділені (різні панелі/кольорові зони).

## 1. Головний екран продажу

```
┌────────────────────────────────────────────────────────────────────────────────────────┐
│ Каса №1 · Склад: Магазин-К1 · Касир: Іваненко І. · Зміна: відкрита 09:02                │
│ Режим: ФІСКАЛЬНИЙ · ФОП: Петренко О.О. (авто) │ ПРРО:🟢 │ Термінал:🟢 │ Принтер:🟢 │ ⚙  │
├────────────────────────────────────────────────────────────────────────────────────────┤
│ [F2] ▌Скан / артикул / назва: _______________________________________________  🔍      │
│      (фокус завжди тут; Enter після скану обробляється автоматично)                    │
├──────────────────────────────────────────────────────────────┬─────────────────────────┤
│ # Товар            ШК        Арт.  К-сть Од  Ціна   Знижка  │ КЛІЄНТ                  │
│                                          Сума  Партія/Серія  │  Роздрібний покупець    │
│ 1 Ніж мисливський  482…901   NK-12  1  шт 1250  −5% акція   │  [F4] знайти/створити   │
│   1187.50          серія: SN-00123 ✅                        │  Бонуси: —              │
│ 2 Ліхтар LED       482…445   LT-03  2  шт  680   —          │─────────────────────────│
│   1360.00          партія B-0524 (до 05.2027) FEFO✅        │ РАЗОМ        2 610.00   │
│ 3 …                                                          │ Знижка         62.50    │
│                                                              │ ДО СПЛАТИ    2 547.50   │
│                                                              │ Отримано      _____     │
│  (повторний скан = +1 к-сть; серійні — окремий рядок)        │ Решта             —     │
├──────────────────────────────────────────────────────────────┴─────────────────────────┤
│ ПРОДАЖ:  [F9 Оплатити] [F5 Клієнт] [F6 Знижка*] [F7 Відкласти] [F8 Повернення]         │
│ ДОКУМЕНТИ: [Ctrl+I Рахунок] [Ctrl+P Оплата рахунку] [Ctrl+V Видача замовлення]         │
│ СКЛАД: [F3 Пошук по складу] [Ctrl+F Всі товари] [Ctrl+R Резерв] [Ctrl+M Переміщення]   │
├─────────────────────────────────────────────────────────────────────────────────────────┤
│ ▓ УПРАВЛІНСЬКА КАСА: [Інкасація] [Витрата] [Внесення] [Касовий звіт] [Товарний звіт]   │
│ ▒ ФІСКАЛЬНЕ МЕНЮ (по ФОП): [X-звіт] [Z-звіт] [Служб.внесення] [Служб.видача] [Статус]  │
└─────────────────────────────────────────────────────────────────────────────────────────┘
```

Правила: після будь-якої завершеної дії фокус → поле швидкого введення; собівартість/маржа
не відображаються (без спец-дозволу); статус-бар оновлюється realtime; помилки — людською мовою
з кодом для адміністратора («ПРРО недоступний: немає з'язку з сервером ДПС. Продаж заблоковано.
[Спробувати ще] [Викликати менеджера]»).

## 2. Вікно оплати (F9)

```
┌── ОПЛАТА · До сплати: 2 547.50 грн · ФОП: Петренко О.О. ────────────────┐
│  [1] Готівка UAH      введено: 2000.00                                  │
│  [2] Картка (термінал) введено:  547.50   → [Надіслати на термінал]     │
│  [3] IBAN переказ  [4] Сертифікат  [5] Бонуси (доступно 120)            │
│  ─────────────────────────────────────────────────────────────          │
│  Разом внесено: 2 547.50   Залишилось: 0.00   Решта: 0.00               │
│  Термінал: очікування підтвердження… ▓▓▓░░  [Скасувати на терміналі]    │
│  ⚠ Змішана оплата: лише фіскальний режим, лише UAH                      │
│  [Enter] Завершити   [Esc] Назад                                        │
└──────────────────────────────────────────────────────────────────────────┘
```

Кнопка «Завершити» заблокована до однозначного результату термінала; при timeout —
автоматичний status-запит, банер «З'ясовуємо статус операції — НЕ повторюйте оплату».

## 3. Відкриття зміни (покупюрний перерахунок)

```
┌── ВІДКРИТТЯ ЗМІНИ · Каса №1 · Іваненко І. ──────────────────────────────┐
│ Валюта: [UAH ▾]  (окремі вкладки: UAH / USD* / EUR*  *якщо увімкнено)   │
│  Номінал   К-сть      Сума        │  1000 ×  3  = 3000.00               │
│  500    ×  4  = 2000.00           │  200  × 10  = 2000.00               │
│  100    ×  5  =  500.00           │  50   ×  4  =  200.00               │
│  20/10/5/2/1/коп …                │                                     │
│  РАЗОМ UAH: 7 700.00                                                     │
│  [Enter] Підтвердити і відкрити зміну     [Esc] Скасувати               │

```

Той самий компонент — для закриття, інкасації, витрат (контекст `context` у DocType).

## 7. Гарячі клавіші (конфігуровані, профіль у POS Cash Desk Profile)

| Клавіша | Дія | | Клавіша | Дія |
|---|---|---|---|---|
| F2 | фокус у швидке введення | | F9 | оплатити |
| F3 | пошук по складу | | F8 | повернення |
| F4/F5 | клієнт | | F7 | відкласти/повернути кошик |
| F6 | знижка (з дозволом) | | Ctrl+I | рахунок |
| ↑/↓, +/− | рядок кошика, к-сть | | Del | видалити рядок |
| Ctrl+Z | скасувати останню дію в кошику | | Esc | закрити вікно |

Зарезервовано: F1 — довідка; Ctrl+Shift+X — фіскальне меню (щоб не натиснути випадково).

## 8. Відкриті UX-питання (до погодження, див. 08)

- кілька паралельних кошиків/«реалізацій» як у Торгсофт: пропозиція — **до 5 відкладених кошиків**
  (Held) з перемикачем вкладок, без повної паралельності;
- звук/колір при помилці скану (товар не знайдено) — пропозиція: звуковий сигнал + червоний банер;
- чи потрібен режим «швидкий продаж без вибору клієнта підтвердженням» — пропозиція: так (Enter-Enter).
